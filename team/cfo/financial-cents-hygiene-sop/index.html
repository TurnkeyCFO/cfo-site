Purpose

Financial Cents is only useful if it reflects reality. Today it drifts: prior-month projects sit open after the books are locked, overdue client reminders pile up, and the team coordinates elsewhere. A tracker that lies about status is worse than no tracker, because people trust it and get burned. This SOP keeps Financial Cents honest for as long as it is in use, and it lays out, neutrally, the open decision on whether Financial Cents or Close Command should be the firm's one workflow system of record.

This SOP does not make that decision. It gives Ricky the usage picture and the tradeoffs so he can.

Owner and roles

Person Role in Financial Cents hygiene
Eldine (Brand Manager and VA) Runs the weekly overdue-reminder triage; owns the routine cleanup of stale reminders and client-task nudges
Vanessa (US Client Lead) Closes prior-month projects at month-lock as part of the close; owns whether a project is genuinely done
Hardik (bookkeeper) Keeps his project task lists current so status pills mean what they say
Ricky (Founder) Owns the system-of-record decision described below

Cadence and triggers

  • At BD10 lock, monthly: the prior month's Financial Cents project for each client is reviewed and closed once the books are locked. A locked month with an open project is a contradiction the close removes.
  • Weekly: Eldine runs the overdue-reminder triage pass across all clients.
  • Ongoing: any project whose status pill no longer matches reality is corrected the day it is noticed, not at month-end.

The process

Close prior-month projects at month-lock

When a client's books lock at BD10, their Financial Cents monthly project for that period is finished work, and the tracker should say so.

The test: at any moment, the set of open Financial Cents projects should equal the set of months actually still in progress. If closed months are showing open, the tracker is drifting.

Weekly overdue-reminder triage

Financial Cents accumulates overdue reminders and client-task nudges. Left alone, the overdue count grows until nobody trusts it. The weekly triage keeps it real.

The system-of-record decision (RICKY DECISION, neutral framing)

There is an open, founder-level decision: should Financial Cents or Close Command be the single workflow system of record? This SOP frames it with the actual usage data and does not decide it.

What the usage data shows. In current operation, Financial Cents Team Chat carries effectively no internal messages; the team coordinates in Slack and works the close in Close Command. Prior-month projects sit open past lock, and overdue reminders accumulate, both of which this SOP exists to fix but which also indicate the tool is not where the team lives day to day. Close Command is the homegrown surface the team actually uses for the close cadence, time tracking, and the transactions review queue.

The two honest options.

  • Recommit to Financial Cents. Keep it as the system of record, and make this hygiene SOP the enforcement so it stops drifting: projects closed at lock, reminders triaged weekly, Team Chat actually used for internal handoffs. The upside is a mature, purpose-built practice-management tool with client-task workflows already in place. The requirement is discipline the team has not yet shown on it, and a recurring per-seat cost that grows with every hire.

  • Make Close Command the system of record. Formalize what is already happening: the team runs the close, tracks time, and reviews transactions in Close Command, and Financial Cents is retired from the workflow role. The upside is standardizing on the tool the team already chooses and uses, at no added per-seat cost. The requirement is building or accepting the parts Financial Cents does today that Close Command does not (client-facing task requests and the client chat surface), and a clean migration of anything Financial Cents currently owns.

The cost consideration at scale. Financial Cents is priced per seat. At the firm's current size the cost is modest, but the scaling plan adds seats, and a per-seat tool that the team routes around becomes a growing line item paying for capability the team is not using. Standardizing on a tool the team avoids will fail quietly regardless of which way the decision goes; the decision should be made before hiring against it.

What this SOP does not do. It does not pick a tool. Until Ricky decides, Financial Cents stays in use and this hygiene routine keeps it honest. If and when the decision lands, this SOP is updated to match.

Guardrails

  • Never leave a locked month's Financial Cents project open. The project reflects the period; a locked period is a closed project.
  • Never let the overdue-reminder list accumulate stale items. Untriaged noise hides the real outstanding work.
  • Never send a team-wide reminder blast for something that concerns one person. Reminders are individualized.
  • Never treat a status pill as true if it has drifted from reality; correct it the day you see it.
  • Never make or announce the system-of-record decision on Ricky's behalf. This SOP frames it; he decides.
  • Keep internal commentary in Team Chat, never in Client Chat, for as long as Financial Cents is in use.

Escalation

Situation What you do
A client's project cannot close because a client item is outstanding Carry it to the next period with a reason; do not leave the locked month open
A client has a chronic overdue-reminder pattern Flag to the QC reviewer or Ricky as a relationship signal, with specifics
The overdue list is unworkable in a weekly pass Raise it at the Monday ops sync; it is a sign the tracker has drifted far enough to need a reset
A question about the system-of-record decision Ricky, always; do not improvise a firm position to a teammate or a client
